# 🔧 Obsidian + WorkBuddy 协作指南

> L2 经验记忆 · 由实际使用中提炼 · 本文档是所有 Obsidian+WorkBuddy 协作经验的汇总

---

## 📐 协作架构

```
你（决策者）
    ↕️ 对话
WorkBuddy（执行层）
    ↕️ 读写文件
Obsidian Vault（D:\Life Knowledge System）
```

WorkBuddy 没有"记忆"，每次新会话都是全新的——**Vault 就是它的外部记忆**。  
你越把关键信息写进 Vault，下次它能接续得越好。

## ⚙️ 已知配置与特殊事项

### WorkBuddy 工作区
- 当前工作区：`c:/Users/Administrator/WorkBuddy/Claw`（WPS集成项目）
- Vault 路径：`D:\Life Knowledge System`
- 两者**不在同一工作区**，WorkBuddy 操作 Vault 文件需要用绝对路径

### 路径注意事项
- Windows 系统，中文路径目录名可以正常使用
- `🧠`、`📥`、`⚙️` 等 emoji 目录名可正常创建
- **不要用脚本批量移动含中文/emoji 的路径**（GBK 编码风险）
