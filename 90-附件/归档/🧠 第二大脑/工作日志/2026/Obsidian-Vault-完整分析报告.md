## 📋 代码优化建议

### 1. Dataview 查询优化

**HOME.md 中的查询**可以简化：
```javascript
// 当前代码
dv.paragraph(`| 快速入口 | 链接 |
|---------|------|
| 📔 今日日记 | [[20-日志/日记/${year}/${month}/${dateStr}\\|${dateStr}]] |
...
`);

// 可以改为使用 dv.table() 更清晰
dv.table(["快速入口", "链接"], [
    ["📔 今日日记", `[[20-日志/日记/${year}/${month}/${dateStr}\\|${dateStr}]]`],
    ...
]);
```

### 2. 日记模板优化

**当前模板**（daily-note-template-simple.md）：
```markdown
## 消费

| 项目 | 金额 |
|------|------|
|  | ¥ |
```

**建议优化**：移除消费板块（因为已独立记账）
```markdown
## 消费

> 消费记录已独立记入 03-领域/财务规划/记账/，此处留空
```

### 3. 领域 README 中的 Dataview 查询

**03-领域-README.md** 中的查询：
```dataview
TABLE status as "健康度", file.mtime as "最近更新"
FROM "03-领域"
WHERE file.name = "领域总览"
SORT status ASC
```

**问题**：`file.name = "领域总览"` 只会匹配 1 个文件，应该改为：
```dataview
TABLE status as "健康度", file.mtime as "最近更新"
FROM "03-领域"
WHERE file.name = "README"
SORT status ASC
```

### 4. 卡片盒统计查询

**10-卡片盒-README.md** 中的查询可能无法工作：
```dataview
TABLE type as "卡片类型", created as "创建时间"
FROM "10-卡片盒"
WHERE file.name != "README"
SORT created DESC
LIMIT 10
```

**问题**：卡片文件可能没有 `type` 属性

**建议**：确保使用模板创建卡片时添加 `type` 属性：
```yaml
---
type: 概念卡片
created: {{date:YYYY-MM-DD}}
---
```

## 📈 使用建议

### 日常使用流程

**每日早晨**：
1. 打开 `00-系统/🏠 HOME.md`
2. 查看"快速入口"，打开今日日记
3. 查看"代办池"，确定今日重点

**每日中午**：
1. 快速回顾上午进展
2. 调整下午计划

**每日晚上**：
1. 更新今日日记（今天/感悟）
2. 消费记录告诉 AI 记入账单
3. 清理收件箱（15分钟）

**每周五晚上**：
1. 写周记（AI 自动生成）
2. 更新各领域健康度
3. 规划下周重点

**每月最后一天**：
1. 写月复盘（AI 自动生成）
2. 更新领域 README 健康度
3. 识别需要改进的地方
4. 制定下月计划

### AI 协作提示词

以下是常用提示词：
- "帮我写今天日记" → 创建当日日记
- "记账：花了XX元，买了XX" → 记入账单
- "帮我写本周周记" → 生成周回顾
- "帮我做本月复盘" → 生成月度复盘
- "建立这个月的账单" → 创建本月账单文件
